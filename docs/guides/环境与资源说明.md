# daily-signal 环境与资源说明

## 1. 文档目的

本说明用于明确本仓库开发所需的本地环境、Python 依赖、外部资源（RSSHub 等）和配置项。

当前目标与设计文档一致：
- 固定时段（08:00 / 17:00）生成本地 Markdown 双文件
- 暂不启用微信推送（仅预留接口）

## 2. 本地开发环境要求

- 操作系统：Windows / macOS / Linux（建议 Windows 11 或 macOS）
- Python：3.11+（建议 3.11）
- 包管理：pip（可选 uv / poetry）
- 数据库：SQLite（内置，无需单独安装）
- 时区：Asia/Shanghai（与计划一致）

可选但推荐：
- Git
- Docker Desktop（如果你要本地自建 RSSHub）

## 3. Python 依赖清单（建议）

> 下面是建议写入 `requirements.txt` 的核心包。

- fastapi：可选管理页面/API
- uvicorn：本地启动 FastAPI
- sqlalchemy：ORM 与 SQLite 访问
- apscheduler：定时任务
- feedparser：解析 RSS/Atom
- jinja2：Markdown 模板渲染
- pydantic：配置与数据校验
- python-dotenv：读取 `.env` 环境变量
- httpx：请求外部接口（可选）
- tenacity：重试与退避（建议）
- markdown：如需网页渲染 Markdown
- pygments：代码高亮（可选）

如果启用 LLM 摘要，再按模型加：
- openai（兼容部分国产模型 OpenAI 风格接口）或各厂商官方 SDK

如果启用语义复筛，再加：
- sentence-transformers（本地向量模型）或相应云 API SDK

## 4. 外部资源说明

### 4.1 RSSHub（必需）

你的这个项目依赖 RSS 源，RSSHub 是核心来源之一。

你有两种使用方式：

1. 直接使用公共实例（最快，无需安装）
- 直接填 `https://rsshub.app/...` 路由
- 优点：零部署
- 缺点：受公共服务稳定性和限流影响

2. 本地自建 RSSHub（推荐长期使用）
- 适合稳定抓取、可控性更高

#### 本地安装方式（Docker，暂时不必须）

前置：已安装 Docker Desktop。

1. 新建 `docker-compose.yml`（可放在任意目录）
2. 填入以下最小配置并启动：

```yaml
version: "3"
services:
  rsshub:
    image: diygod/rsshub
    container_name: rsshub
    restart: unless-stopped
    ports:
      - "1200:1200"
    environment:
      NODE_ENV: production
```

启动命令：

```bash
docker compose up -d
```

验证：
- 浏览器打开 `http://localhost:1200`
- 测试路由示例：`http://localhost:1200/github/trending/daily`

项目中对应 `rss_url` 可改为：
- `http://localhost:1200/zhihu/hotlist`
- `http://localhost:1200/github/trending/daily`
- 其他路由按 RSSHub 文档填写

### 4.2 opencli（当前阶段非必需）

结论：当前设计不要求 `opencli`，可以不安装。

适用场景（可选）：
- 你希望通过统一 CLI 管理部分抓取或自动化流程时再评估引入。

## 5. 你需要配置的内容（最小必配）

按 `config.yaml` 规划，至少要配置：

1. `schedule`
- 时区：`Asia/Shanghai`
- 固定时段：`08:00`（AM）和 `17:00`（PM）
- `compensate_on_startup: true`

2. `sources`
- 你关心的平台路由（twitter/zhihu/bilibili/weibo/github）
- 每个源的 `rss_url`、`enabled`、`source_weight`

3. `filters`
- 黑名单词
- 黑名单域名
- 最小正文长度
- 分数阈值

4. `output`
- 输出目录（默认 `./output`）

可选配置：
- `summarizer`（启用 LLM 时配置 API Key）
- `push`（当前可保持 `enabled: false`）

## 6. 环境变量建议（`.env`）

如果启用摘要/推送，建议在 `.env` 中配置：

```env
DEEPSEEK_API_KEY=
SERVERCHAN_KEY=
PUSHPLUS_TOKEN=
```

当前阶段只做本地 Markdown，可先不填。

## 7. 快速启动建议（开发期）

1. 创建并激活虚拟环境
2. 安装依赖
3. 准备 `config.yaml`（先用 1-2 个 source）
4. 使用公共 RSSHub 路由先跑通
5. 稳定后再切本地 RSSHub（Docker）

## 8. 常见问题

1. 早上没开机导致 08:00 没跑
- 解决：开机补偿执行 + 状态持久化 + 幂等控制

2. RSSHub 路由偶发失败
- 解决：增加抓取重试、超时、退避；必要时自建 RSSHub

3. 同一内容重复出现在日报
- 解决：canonical_url + title_fingerprint 双重去重
